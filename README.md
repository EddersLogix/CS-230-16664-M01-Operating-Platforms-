# CS-230-16664-M01-Operating-Platforms-
Portfolio
The Gaming Room is a client that originally built Draw It or Lose It as an Android‑only game and needed a scalable, secure, multiplatform web application capable of supporting thousands of concurrent players. Their requirements centered on cross‑platform accessibility, consistent game state management, secure communication, and unique identifiers for all game entities. In developing the software design documentation, I did particularly well in connecting object‑oriented principles with operating‑system concepts such as process isolation, protection domains, and resource control, which helped justify architectural choices like using the Singleton and Iterator patterns. Working through the design document made the coding process more structured because defining the domain model, constraints, and architecture upfront gave me a clear blueprint for implementation. If I were to revise one part of the document, I would expand the System Architecture View with a visual diagram to make the design more accessible to stakeholders and strengthen the connection between the written architecture and the deployment model. I interpreted the user’s needs by focusing on scalability, security, and platform independence, ensuring the design addressed real client requirements rather than just technical preferences; considering user needs is essential because it leads to systems that are usable, reliable, and aligned with real‑world expectations. My overall approach to designing software relied on iterative analysis, UML modeling, design patterns, and OS‑level reasoning, and in future projects I would continue using these strategies while incorporating early prototyping, containerization, and cloud deployment planning to validate scalability and reliability earlier in the design process.
